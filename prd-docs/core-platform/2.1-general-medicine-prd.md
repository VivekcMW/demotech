## 1. GOAL

Deliver a production-ready **General Medicine/Internal Medicine** specialty implementation that works across:
- solo doctor clinic,
- multi-doctor clinic,
- single-branch hospital,
- multi-branch chain.

---

## 2. RELEASE OBJECTIVES

### R1 (MVP)
- End-to-end OPD flow (new + follow-up)
- Core GenMed templates
- Core order sets (fever, HTN, diabetes)
- E-prescription + basic chronic registry prompts

### R2
- IPD workflow (admission, progress rounds, discharge)
- Enhanced chronic disease management and care-gap alerts

### R3
- Multi-branch governance and benchmarking
- Advanced recall and outcomes workflows

## 3. EPICS & USER STORIES

## Epic GM-01: GenMed Consultation Templates

### Story GM-01-01
**As a** GenMed doctor, **I want** a structured new-consult template **so that** documentation is complete and quick.

**Acceptance Criteria**
1. Template includes mandatory fields: chief complaint, history, exam, assessment, plan.
2. Note cannot be signed without primary ICD diagnosis.
3. Autofill available for demographics/allergy/problem list.

### Story GM-01-02
**As a** doctor, **I want** follow-up templates with copy-forward controls **so that** repeat visits are faster without unsafe stale text.

**Acceptance Criteria**
1. Copy-forward sections are visibly marked.
2. User confirmation required before sign.
3. Stale section warning shown if prior note age exceeds configured threshold.

---

## Epic GM-02: Chronic Disease Longitudinal Care

### Story GM-02-01
**As a** physician, **I want** automatic chronic registry enrollment **so that** continuity care is trackable.

**Acceptance Criteria**
1. Registry enrollment auto-triggers on configured chronic ICD codes.
2. Registry tag appears on patient summary.
3. Registry can be manually overridden with reason.

### Story GM-02-02
**As a** doctor, **I want** care-gap prompts **so that** overdue monitoring is not missed.

**Acceptance Criteria**
1. Diabetes care-gap alert appears when HbA1c overdue > configured period.
2. HTN uncontrolled warning appears after configured threshold visits.
3. Alert acknowledgement logged.

---

## Epic GM-03: GenMed Order Sets and CPOE Optimization

### Story GM-03-01
**As a** doctor, **I want** GenMed order sets (fever, HTN, DM, dyspnea) **so that** ordering is standardized.

**Acceptance Criteria**
1. Order sets launch from consult note in <= 2 clicks.
2. Safety checks (allergy/interaction/duplicate) execute before sign.
3. Items can be toggled on/off before final sign.

### Story GM-03-02
**As a** clinician, **I want** stat pathway support for acute cases **so that** urgent care is expedited.

**Acceptance Criteria**
1. Stat orders routed with high-priority flag.
2. Critical result alerts sent to ordering clinician and assigned nurse.
3. Acknowledgment SLA visible and reportable.

---

## Epic GM-04: Multi-Doctor Operations

### Story GM-04-01
**As an** operations lead, **I want** pooled and doctor-specific queue modes **so that** clinics can run different staffing models.

**Acceptance Criteria**
1. Queue mode configurable by facility/department.
2. Reassignment between doctors retains full audit trail.
3. Wait-time impact displayed when reassigned.

### Story GM-04-02
**As a** covering doctor, **I want** handover summaries **so that** I can safely continue care.

**Acceptance Criteria**
1. Last assessment, active meds, pending tests shown in compact handover view.
2. Missing critical fields block handover completion.
3. Handover sign-off by sender/receiver tracked.

---

## Epic GM-05: IPD GenMed Workflow

### Story GM-05-01
**As a** duty physician, **I want** admission assessment template **so that** inpatient initiation is standardized.

**Acceptance Criteria**
1. Admission template captures diagnosis, severity, initial plan.
2. Initial order bundle can be applied from template.
3. Admission note signed before bed transfer finalization.

### Story GM-05-02
**As a** consultant, **I want** daily progress and discharge hard-stops **so that** documentation quality remains compliant.

**Acceptance Criteria**
1. Daily progress note expected by configured cutoff time.
2. Discharge summary requires final diagnosis + med reconciliation + follow-up date.
3. Discharge note immutable post-sign except addendum.

---

## Epic GM-06: Multi-Branch Governance

### Story GM-06-01
**As a** central medical governance lead, **I want** branch template governance **so that** quality is consistent.

**Acceptance Criteria**
1. Central templates publish with maker-checker approval.
2. Branch override requests require approval and reason.
3. Effective dates and rollback available.

### Story GM-06-02
**As a** chain clinician, **I want** cross-branch continuity access **so that** follow-up is seamless.

**Acceptance Criteria**
1. Cross-branch chart access follows consent + RBAC scope.
2. Access events logged as audit entries.
3. Cross-branch referral note links source and destination encounters.

---

## Epic GM-07: Reporting & KPI Layer

### Story GM-07-01
**As an** admin, **I want** operational GenMed dashboards **so that** throughput and quality are visible.

**Acceptance Criteria**
1. KPIs include OPD TAT, no-show rate, follow-up adherence.
2. Filters by doctor, branch, date range available.
3. Export to PDF/CSV available.

### Story GM-07-02
**As a** quality lead, **I want** safety/compliance dashboards **so that** risks are detected early.

**Acceptance Criteria**
1. Critical result acknowledgment compliance shown.
2. Documentation deficiency trend shown.
3. Readmission and escalation metrics shown.

## 6. UAT CHECKLIST

1. New consult can be completed in < 5 minutes with structured template.
2. Follow-up consult can reuse prior context safely.
3. Order sets place all expected orders correctly.
4. Critical result acknowledgment path works with escalation.
5. Solo and multi-doctor queue modes both pass scenario tests.
6. IPD admission and discharge notes satisfy mandatory fields.
7. Cross-branch access obeys consent + RBAC.
8. KPI values match transactional records.

---

## 7. RISKS & MITIGATIONS

| Risk | Impact | Mitigation |
|---|---|---|
| Template overload (too many mandatory fields) | Slower doctor adoption | Progressive mandatory model + specialty review |
| Alert fatigue from CDS | Ignored warnings | Severity tiering + tuning governance |
| Cross-branch data access concerns | Compliance risk | Strict scope model + audit alerts |
| Inconsistent branch operations | KPI variance | Central template governance + local override policy |

---

## 8. DEFINITION OF DONE

A backlog item is “Done” only when:
1. Functional acceptance criteria pass.
2. Audit logging for key actions verified.
3. Role-based access checks validated.
4. Documentation and dashboard impacts tested.
5. UAT sign-off obtained from clinical + operations stakeholders.
