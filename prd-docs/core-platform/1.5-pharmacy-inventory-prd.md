# PRD 1.5 — Pharmacy & Inventory
## Indian EHR System — Medication Fulfillment and Stock Control

**Document ID:** PRD-CORE-1.5  
**Version:** 1.0  
**Last Updated:** June 10, 2026  
**Status:** Draft

## 1. Overview

Defines unified pharmacy workflows for OPD/IPD/daycare and inventory control for medicines, consumables, implants, and high-alert drugs.

### Scope
- eRx intake from CPOE
- Pharmacist verification and dispensing
- IPD ward issue and returns
- Formulary and substitution rules
- Batch/expiry tracking
- Purchase → GRN → stock → issue lifecycle
- Controlled drug registers
- Alerts (stockout, near expiry, cold chain deviation)

## 2. Personas
- Pharmacist
- Pharmacy in-charge
- Doctor
- Nurse
- Procurement officer
- Finance/admin

## 3. Functional Requirements

### 3.1 Prescription Fulfillment
- Receive eRx from `1.4` with full metadata
- Validate dosage, interactions, allergies, contraindications
- Dispense full/partial with backorder logic
- Capture substitution proposal + prescriber approval where required
- Print/share label with dose instructions and language preference

### 3.2 Inpatient Medication Supply
- Ward stock indents and floor-stock replenishment
- Patient-specific issue linked to MAR tasks
- Return/reversal with reason and audit
- Emergency issue workflow (code blue, OT urgent)

### 3.3 Inventory Core
- Item master: generic/brand, strength, form, pack size, schedule class
- Multi-location stock bins (main store, satellite pharmacy, OT store)
- Batch and expiry mandatory for medicines
- FEFO picking policy (First Expiry First Out)
- Reorder point and min-max policy per location

### 3.4 Procurement & Receiving
- Purchase order creation and approval workflow
- Goods receipt note (GRN) with invoice and batch capture
- Short/extra/damaged receipt handling
- Vendor return workflow

### 3.5 Controlled Substances
- Separate register for Schedule X/H1 or configured lists
- Dual authorization for issue/correction
- Daily reconciliation and variance alerts

### 3.6 Cold Chain
- Temperature logs for cold storage items
- Excursion alert with quarantine workflow

## 4. Data Model (Core Entities)
- `DRUG_MASTER`
- `INVENTORY_ITEM`
- `STOCK_BATCH`
- `STOCK_LEDGER`
- `PRESCRIPTION_FULFILLMENT`
- `WARD_ISSUE`
- `PURCHASE_ORDER`
- `GRN`
- `CONTROLLED_DRUG_REGISTER`

## 5. Workflows
1. eRx → pharmacist verify → dispense → bill event
2. IPD order → ward issue → administration linkage → return/reconcile
3. PO → GRN → stock ledger update

## 6. Integrations
- `1.4 Orders & CPOE` (inbound medication orders)
- `1.8 Nursing` (administration status)
- `1.6 Billing` (charge capture)
- Optional external pharmacy POS/ERP

## 7. Reports
- Dispensing TAT
- Stock aging and near-expiry
- Stockout incidents
- Controlled drug variance
- Generic substitution rate

## 8. NFRs
- Dispense action latency < 2 sec
- Stock query < 300 ms p95
- 100% ledger auditability

## 9. Acceptance Criteria
1. Medication orders flow from CPOE and are fulfillable end-to-end
2. Batch/expiry tracked for all dispensed medicines
3. Controlled drug dual-auth enforced
4. Stock ledger updates for every movement
5. FEFO applied by default with override reason capture
6. Near-expiry alerts configurable and functional
