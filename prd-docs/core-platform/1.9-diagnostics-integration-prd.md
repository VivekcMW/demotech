# PRD 1.9 — Diagnostics Integration (LIS/RIS/PACS)
## Indian EHR System — Lab and Imaging Interoperability

**Document ID:** PRD-CORE-1.9  
**Version:** 1.0  
**Last Updated:** June 10, 2026

## 1. Overview

Defines diagnostic interoperability for laboratory and radiology services, enabling order transmission, worklist orchestration, result ingestion, critical value escalation, and clinical consumption.

### Scope
- LIS integration for lab orders/results
- RIS/PACS integration for imaging workflow/report status
- HL7/FHIR/DICOM compatibility requirements
- Specimen and modality worklists
- Critical value and acknowledgement loops

## 2. Personas
- Lab technician
- Pathologist
- Radiology technician
- Radiologist
- Ordering clinician
- IT integration admin

## 3. Functional Requirements

### 3.1 Order Interface
- Receive orders from CPOE and route to LIS/RIS queues
- Message retry and idempotency controls
- Priority propagation (routine/urgent/stat)

### 3.2 Lab Workflow Integration
- Order accepted → sample collected → sample received → processing → verified → released
- Specimen barcode mapping
- Rejection/recollection status support

### 3.3 Radiology Workflow Integration
- Order accepted → scheduled/performed → reporting → finalized
- DICOM study reference linkage
- Report status sync back to EHR

### 3.4 Result Ingestion
- Structured lab values, units, flags
- Radiology report text and attachments
- Versioned result updates/corrections

### 3.5 Critical Result Logic
- Configurable critical thresholds
- Immediate alerts to ordering team
- Mandatory acknowledgment trail

### 3.6 Downtime and Reconciliation
- Queue buffering during external system downtime
- Replay and reconciliation tools for missed events

## 4. Data Model
- `DIAG_ORDER_XREF`
- `LIS_MESSAGE_LOG`
- `RIS_MESSAGE_LOG`
- `RESULT_RECORD`
- `RESULT_VERSION`
- `CRITICAL_ALERT_EVENT`
- `ACK_EVENT`

## 5. Workflows
1. CPOE order → external system ack → execution updates → result return
2. Critical value return → escalation → acknowledgment capture
3. Downtime queue replay and exception closure

## 6. Integrations & Standards
- HL7 v2.x messages (where used)
- FHIR API adapters (where supported)
- DICOM/PACS references for imaging
- Interface monitoring and dead-letter queue

## 7. Reports
- Diagnostic TAT by test/modality
- Critical value acknowledgment compliance
- Integration error and retry analytics

## 8. NFRs
- Result ingestion latency < 60 sec typical
- Interface success rate > 99%
- Complete traceability for each order-result pair

## 9. Acceptance Criteria
1. Orders consistently reach LIS/RIS and status syncs back
2. Lab/radiology results linked to originating order without mismatch
3. Critical values trigger alert and acknowledgment workflow
4. Integration downtime recovery can replay missed messages safely
5. Interface logs support full troubleshooting trace
