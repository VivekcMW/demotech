# PRD 1.7 — Bed & ADT Management
## Indian EHR System — Admission, Transfer, Discharge Control

**Document ID:** PRD-CORE-1.7  
**Version:** 1.0  
**Last Updated:** June 10, 2026

## 1. Overview

Defines inpatient flow management from admission request to discharge, including bed allocation, transfers, occupancy visibility, and discharge readiness.

### Scope
- Admission requests (OPD/ER/direct)
- Bed board and occupancy tracking
- Transfer workflows (ward/ICU/room/bed)
- Discharge planning and clearance states
- Bed housekeeping and turnaround
- Reservation and block rules

## 2. Personas
- Admission desk
- Bed manager
- Ward nurse
- Duty doctor
- Housekeeping supervisor
- Billing/discharge desk

## 3. Functional Requirements

### 3.1 Admission
- Create admission case linked to patient and primary consultant
- Capture admission type: elective/emergency/daycare conversion
- Admission priority and required bed class
- Pre-admission checklist support

### 3.2 Bed Board
- Real-time bed state: vacant/occupied/reserved/blocked/cleaning/maintenance
- Filters by facility, block, ward, specialty, bed class
- Isolation bed tagging and gender constraints

### 3.3 Bed Allocation
- Rule-based assignment by specialty, acuity, payer, bed class
- Override with reason and approval
- Auto-suggest nearest available compliant bed

### 3.4 Transfer Management
- Inter-ward and ICU transfer requests
- Clinical and billing timestamp capture
- Handover dependency checks before transfer complete

### 3.5 Discharge Workflow
- Medical discharge order
- Nursing clearance
- Pharmacy return/clearance
- Billing clearance
- Final discharge timestamp and bed release to cleaning

### 3.6 Bed Turnaround
- Cleaning task auto-created on discharge/transfer out
- SLA tracking (e.g., room ready in X minutes)
- Escalation on delay

## 4. Data Model
- `ADMISSION_CASE`
- `BED_MASTER`
- `BED_STATUS_EVENT`
- `BED_ASSIGNMENT`
- `TRANSFER_EVENT`
- `DISCHARGE_CASE`
- `BED_CLEANING_TASK`

## 5. Workflows
1. Admission request → bed allocation → occupancy update
2. Transfer request → handover → bed swap and status updates
3. Discharge order → multi-team clearance → bed cleaning → available

## 6. Integrations
- `1.2` Scheduling (planned admissions)
- `1.6` Billing (room/bed charges)
- `1.8` Nursing (handover/clearance)
- `1.5` Pharmacy (medication/device returns)

## 7. Reports
- Occupancy % by unit
- Average length of stay
- Bed turnaround time
- Transfer patterns and ICU utilization

## 8. NFRs
- Bed board refresh < 3 sec
- Allocation suggestion < 1 sec
- 99.9% availability

## 9. Acceptance Criteria
1. Real-time bed status accurately reflects events
2. Admission-to-bed assignment auditable end-to-end
3. Transfers retain both clinical and financial timestamps
4. Discharge cannot finalize without required clearances
5. Bed cleaning SLA breaches are alertable and reportable
