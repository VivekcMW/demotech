## The problems we built around
- **Skin is visual; records usually aren't:** Without standardized photos, neither treatment response nor disputes can be assessed.
- **Aesthetics runs on packages and sessions:** Six-session laser courses need scheduling, balance tracking, and revenue recognition.
- **Consent and expectation management:** Cosmetic procedures invite disputes that only documentation defuses.

## How dermatology runs on AarogyaEHR
### Photo documentation
Standardized-angle capture from tablets, side-by-side timeline comparisons, annotation, and consent-controlled usage flags.
### Session package engine
Multi-session packages (laser/PRP/peels) with per-session redemption, parameter logging (fluence, depth), and balance visibility for the patient.
### Clinical dermatology packs
Psoriasis (PASI scoring), acne grading, vitiligo mapping, biologic-therapy monitoring with lab safety panels.

Procedure-specific consent with realistic-outcome language, bilingual, photo-linked.
